**Issue:** ELI-45  
**Author:** CTO Agent

**Status:** Proposed

## 1. Executive Summary

The current trading bot generates signals from three strategies (momentum, scalp, mean-reversion) with hand-tuned thresholds that adapt only via a simple win-rate heuristic. This design adds a lightweight ML layer that:

1. **Tracks all signals** (not just executed trades) with their indicators, strategy, and eventual outcome
2. **Scores signal confidence** using a logistic regression model trained on historical accuracy
3. **Filters weak signals** and **boosts strong ones** based on ML confidence
4. **Auto-tunes strategy parameters** (RSI thresholds, EMA periods, signal thresholds) using a bandit-based optimizer
5. **Targets**: improve win rate from ~50% baseline to 60%+

All inference runs inside Netlify serverless functions (under 10ms per signal), with training done offline.

## 3. ML Architecture

### 3.1 Model Choice: Logistic Regression

**Why not a neural net?**
- Must run inside Netlify serverless (50ms function start + 10ms inference budget)
- Logistic regression inference is O(features) — ~10 microsecond per signal
- Training requires only ~100-500 samples (not thousands)
- We have structured numeric indicators, not images/text
- Logistic regression is interpretable — we can inspect which features matter

**Feature engineering** (14 features per signal):

| # | Feature | Source | Rationale |
|---|---------|--------|-----------|
| 1 | signal_strength | strategy.strength | Raw signal confidence |
| 2 | rsi | indicators.rsi | Overbought/oversold |
| 3 | atr_pct | indicators.atrPct | Volatility regime |
| 4 | bb_percent_b | indicators.bbPercentB | Bollinger position |
| 5 | ema_spread | (ema9-ema21)/price | Trend direction |
| 6 | volume_ratio | indicators.volumeRatio | Volume confirmation |
| 7 | daily_change_pct | indicators.dailyChangePct | Momentum |
| 8 | is_momentum | one-hot(strategy) | Strategy type |
| 9 | is_scalp | one-hot(strategy) | Strategy type |
| 10 | is_mean_reversion | one-hot(strategy) | Strategy type |
| 11 | is_confirmed | confirmed ? 1 : 0 | Multi-TF confirmation |
| 12 | num_reasons | reasons.length | Signal clarity |
| 13 | macd_hist | indicators.macdHistogram | Momentum oscillator |
| 14 | holding_period_estimate | 1/atr_pct | Expected holding period |

**Label**: binary — `1` if the trade P&L was positive (win), `0` if negative (loss) or expired.

### 3.2 Normalization

All features are z-score normalized using a running mean/std computed during training:

```
z_feature = (feature - mean) / std
```

The mean and std are stored in `MLModel.scaler` and updated during retraining.

### 3.3 Prediction

```javascript
function predict(features, model) {
  const z = features.map((f, i) => (f - model.scaler.mean[i]) / model.scaler.std[i]);
  const logit = model.bias + z.reduce((sum, zi, i) => sum + zi * model.weights[i], 0);
  return 1 / (1 + Math.exp(-logit)); // sigmoid
}
```

**Cost**: 14 multiplications + 1 sigmoid = negligible even in a cold-started serverless function.


### 4.2 Training Trigger

Training runs **inside the bot** every 24 hours (or when signal log exceeds 200 records):

```javascript
// In trading-bot-v5.mjs, at the end of each run:
if (shouldRetrain(signalLogCount, hoursSinceLastTraining)) {
  await retrainModel();
}
```

### 4.3 Training Algorithm

Simple gradient descent on cross-entropy loss, runs in <100ms in Node.js:

```javascript
// Logistic regression training (gradient descent)
function train(features, labels, epochs = 100, lr = 0.01) {
  const n = features.length;
  const nFeatures = features[0].length;
  
  // Compute normalization stats
  const mean = new Array(nFeatures).fill(0);
  const std = new Array(nFeatures).fill(1);
  for (let j = 0; j < nFeatures; j++) {
    const vals = features.map(f => f[j]);
    mean[j] = vals.reduce((a, b) => a + b) / n;
    std[j] = Math.max(0.001, Math.sqrt(vals.reduce((a, b) => a + (b - mean[j]) ** 2, 0) / n));
  }
  
  // Normalize
  const X = features.map(f => f.map((v, j) => (v - mean[j]) / std[j]));
  
  // Initialize weights
  let weights = new Array(nFeatures).fill(0);
  let bias = 0;
  
  // Gradient descent
  for (let epoch = 0; epoch < epochs; epoch++) {
    let gradW = new Array(nFeatures).fill(0);
    let gradB = 0;
    
    for (let i = 0; i < n; i++) {
      const logit = bias + X[i].reduce((s, x, j) => s + x * weights[j], 0);
      const pred = 1 / (1 + Math.exp(-logit));
      const error = pred - labels[i];
      
      for (let j = 0; j < nFeatures; j++) {
        gradW[j] += error * X[i][j];
      }
      gradB += error;
    }
    
    for (let j = 0; j < nFeatures; j++) {
      weights[j] -= lr * (gradW[j] / n + 0.01 * weights[j]); // L2 regularization
    }
    bias -= lr * (gradB / n);
  }
  
  return { weights, bias, scaler: { mean, std } };
}
```

### 4.4 Model Evaluation

After training, evaluate on held-out 20% and store metrics in `MLModel.trainingStats`. Only deploy new weights if:

- Accuracy > 55% (better than random)
- AUC > 0.6
- At least 30 evaluation samples
- Win-rate on evaluation set > current bot win rate

If model fails validation, keep the previous model.

## 5. Inference API

### 5.1 Signal Scoring

```javascript
// In trading-bot-v5.mjs, after generating signals:
for (const signal of signals) {
  const features = extractFeatures(signal);
  const mlScore = predict(features, mlModel);
  signal.mlConfidence = mlScore;
  
  // Filter: skip signals below ML threshold
  if (mlScore < adaptiveParams.mlConfidenceThreshold) {
    log(`[ML] Skipping ${signal.symbol}: ML confidence ${mlScore.toFixed(2)} < threshold ${adaptiveParams.mlConfidenceThreshold}`);
    continue;
  }
  
  // Boost: scale position size for high-confidence signals
  if (mlScore > 0.75) {
    signal.positionSizeMultiplier = adaptiveParams.mlBoostFactor;
    log(`[ML] Boosting ${signal.symbol}: ML confidence ${mlScore.toFixed(2)}`);
  }
  
  // Execute with ML-modified signal
  await executeSignal(signal, riskManager, equity, currentPositions);
}
```

### 5.2 Strategy Weighting

ML confidence also per-strategy:

```javascript
// After accumulating 100+ signals per strategy:
signal.strength *= adaptiveParams.strategyWeights[signal.strategy] || 1.0;
```

Strategy weights are updated during param tuning (Section 6).

## 6. Adaptive Parameter Tuning

### 6.1 Thompson Sampling Bandit

Each tunable parameter has a range and a Beta distribution (alpha, beta) representing our belief about its optimal value.

```typescript
interface ParamBandit {
  name: string;
  range: [number, number];        // Min/max allowed values
  alpha: number;                  // Beta distribution alpha (successes + 1)
  beta: number;                    // Beta distribution beta (failures + 1)
  current: number;                // Current value being tested
  results: Array<{value: number, win: boolean}>;  // History
}
```

Every 24 hours:
1. **Sample** a candidate value from each parameter's Beta distribution
2. **Evaluate** the bot's performance with current params (win rate over last 50 trades)
3. **Update** the Beta distribution: if win rate improved, increase alpha; else increase beta
4. **Gradually converge** toward the best-performing values

### 6.2 Tunable Parameters

| Parameter | Min | Max | Step | Current Default |
|-----------|-----|-----|------|-----------------|
| signalThreshold | 0.10 | 0.40 | 0.01 | 0.22 |
| scalpThreshold | 0.08 | 0.25 | 0.01 | 0.15 |
| rsiOversold | 20 | 40 | 1 | 35 |
| rsiOverbought | 60 | 80 | 1 | 65 |
| volumeMultiplier | 1.0 | 2.0 | 0.1 | 1.2 |
| mlConfidenceThreshold | 0.45 | 0.70 | 0.01 | 0.55 |
| mlBoostFactor | 1.0 | 2.0 | 0.1 | 1.3 |

### 6.3 Safety Constraints

- No parameter change may increase maximum daily trade count by >50%
- No parameter change may increase max position size by >2x
- Win rate below 40% triggers parameter rollback to last known-good values

## 9. Risk Assessment

| Risk | Mitigation |
|------|-----------|
| Overfitting to recent data | L2 regularization, minimum 100 samples before training, 20% holdout |
| Model drift (market regime change) | Retrain every 24h, fallback to baseline if accuracy drops |
| Cold start (no model) | Start with all-zero weights = no ML effect, gradual ramp-up |
| Blobs storage growth | Cap signal log at 10,000 records, FIFO eviction |
| Training latency | Logistic regression trains in <100ms, no impact on bot cycle |
| Wrong parameter tuning direction | Thompson sampling with safety constraints, rollback on <40% win rate |
